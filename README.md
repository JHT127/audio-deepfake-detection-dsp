# Audio Deepfake Detection Using Classical DSP

Detecting AI-generated/spoofed speech using classical digital signal
processing techniques (FFT, filtering, spectral feature extraction) —
without relying on black-box deep learning.

Reproduces and extends the feature pipeline from:
Alzantot et al., "Deep Residual Neural Networks for Audio Spoofing
Detection," Interspeech 2019.

## Using the preprocessing pipeline (for Persons B and C)

The preprocessing module (`src/preprocessing.py`) is the shared entry
point for going from a raw audio file to frequency-domain features.
Build on top of this rather than re-implementing framing/FFT logic.

**Public interface:**

```python
load_and_frame(filepath) -> frames, shape (num_frames, 400)
    # 400 = 25ms @ 16kHz, already Hamming-windowed, 50% overlap.

get_spectrogram(frames)      -> magnitude spectrogram, shape (num_frames, 201)
get_log_spectrogram(frames)  -> log-magnitude spectrogram, same shape
    # 201 = 400 // 2 + 1 rfft bins.
```

### Pre-computed frames are already on disk

`scripts/process_dataset.py` has already run over all 300 files. For
each raw file `data/raw/<label>/<name>.flac`, there is a corresponding
`data/processed/<label>/<name>.npy` containing the **windowed frames**
(output of `load_and_frame()`), shape `(num_frames, 400)`.

> **Important:** these `.npy` files are raw frames, not spectrograms.
> If you need frequency-domain features, load the frames and call
> `get_spectrogram()` / `get_log_spectrogram()` yourself.

### Train/test split

`data/split.csv` is the shared manifest, with columns:

| column   | meaning                        |
| -------- | ------------------------------ |
| filename | `<name>.npy`, no folder prefix |
| label    | `real` or `fake`               |
| split    | `train` or `test`              |

To find the actual array on disk for a row, join `label` and `filename`:
`data/processed/{label}/{filename}`.

### Typical usage pattern

```python
import pandas as pd
import numpy as np
from src.preprocessing import get_spectrogram, get_log_spectrogram

split_df = pd.read_csv("data/split.csv")

for _, row in split_df.iterrows():
    frames = np.load(f"data/processed/{row.label}/{row.filename}")  # (num_frames, 400)

    log_spec = get_log_spectrogram(frames)   # (num_frames, 201) — for STFT-style features
    # or extract other DSP features directly from `frames` (zero-crossing
    # rate, short-time energy, autocorrelation, etc. — Task 5)

    is_train = row.split == "train"
    label = row.label  # "real" or "fake" — map to 0/1 as needed for your classifier
```

### If you need a fresh signal instead of precomputed frames

Use `load_and_frame(filepath)` directly on any file in
`data/raw/<label>/` — same output as what's stored in
`data/processed/`, useful if you want a different `window` type or
`apply_preemph=True` for experimentation (Task 6 filter comparisons,
Task 8 improvements).
